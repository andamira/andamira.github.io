pub struct NodeUsize<D = (), const LCAP: usize = 0> {
    pub data: D,
    pub links: [Option<NonExtremeUsize>; LCAP],
}
Available on crate feature _node_usize only.
Expand description

A generic node with a configurable capacity for links.

It’s designed to be used in graphs, linked lists, and other node-based data structures.

§Data

If D = () the data field will occupy 0 memory.

§Links

Link values can range from 0..usize::MAX (exclusive), and have the same memory layout optimization as a NonZeroUsize.

The LCAP const-generic argument determines the capacity for links. If LCAP == 0 the [link][NodeUsize.link] field will occupy 0 memory.

Links have a unique local ID associated corresponding to their index in the link field array.

A link ID can range from 0..min(LCAP, usize::MAX) (exclusive).

The provided methods for links use usize instead of NonExtremeUsize, for convinience. It’s also possible to access the links field directly.

§Features

It is implemented for the corresponding link-sizes by enabling the features: _node_u[8|16|32|64|128].

§Methods

  • Construct: new(uc), empty,
  • data_ref, data_mut,
  • replace_data

Fields§

§data: D

The node data. Defaults to the unit type if not specified.

§links: [Option<NonExtremeUsize>; LCAP]

An array of optional node links. Capacity defaults to 0 if not specified.

pub fn add_link_unique(&mut self, link: usize) -> Result<usize>

Adds the given link to the node, but only if it’s unique.

It calls find_link to search for duplicates.

§Errors

Returns OutOfBounds if link >= LCAP NodeLinkNotUnique if the link is not unique.

§Examples
use devela::data::{NodeUsize as Node, DataError};

let mut n = Node::<(), 3>::new().unwrap();
assert![n.add_link_unique(0).is_ok()];
assert![n.add_link_unique(1).is_ok()];
assert![n.add_link_unique(2).is_ok()];

assert_eq![n.add_link_unique(1), Err(DataError::NodeLinkNotUnique(Some(1)))];
assert_eq![n.add_link(3), Err(DataError::OutOfBounds(Some(3)))];
