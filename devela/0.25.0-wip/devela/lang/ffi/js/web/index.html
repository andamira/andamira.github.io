§FFI Handles to JavaScript Objects

This module provides Rust wrappers for JavaScript objects (e.g. WebWorker, WebElement…) These types are non-owning handles representing lightweight references to objects managed entirely by JavaScript. Their semantics are as follows:

  • Cheap to copy: All handles are Copy + Clone because they internally store only an ID (e.g. js_uint32). Passing them by value is zero-cost and more efficient than references.

  • No ownership or lifecycle control: JavaScript retains full ownership. Rust handles are merely identifiers, and their existence does not guarantee the underlying JS object is alive.

    • Example: A WebWorker may be terminated by JS, making the handle “stale”.

  • Validity checks: Use methods like is_active() (if available) or error handling (e.g., WebWorkerError::WorkerNotFound) to detect stale handles.

  • Thread safety: Handles are Send + Sync if applicable, but all FFI operations must still occur on the main thread (or the thread where the JS object is valid).

  • FFI transparency: Handles are #[repr(transparent)] when possible over their integer ID for seamless WASM/JavaScript interoperability.
