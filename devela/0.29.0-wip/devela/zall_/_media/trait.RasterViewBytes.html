Expand description

🖼️ A borrowed dense 2D byte raster view with explicit row layout.

📍 media/visual/image/raster

This is the safe byte-first bridge for presentation and backend upload.

It is for rasters whose consumer needs raw bytes and explicit scanline layout. It is a sibling of RasterView, not a replacement for it.

Required Methods§

Source

fn raster_extent_bytes(&self) -> Extent2<u32>

Returns the logical raster extent in samples or pixels.

This uses a distinct method name to avoid ambiguity with RasterView on types that implement both traits.

Source

fn raster_depth(&self) -> u8

Returns the logical raster depth in bits.

This is a compact backend-style depth value. It may differ from the stored bits per pixel. For example, a 32-bit stored XRGB pixel may have depth 24.

Source

fn raster_bytes(&self) -> &[u8] ⓘ

Returns the raw raster bytes.

Source

fn raster_bytes_per_line(&self) -> usize

Returns the stored bytes per scanline.

fn raster_bits_per_pixel_bytes(&self) -> Option<u16> ⓘ

Returns the stored bits per pixel, if known from the row layout.

This is storage width, not logical depth.
