Random number generation.

rand ◉

Random sources, value construction, qualities, and pseudo-random generators.

§Traits

  • RandTry and Rand define fallible and infallible random sources.
  • RandQualities describes source behavior and suitability.
  • FromRandTry and FromRand construct random values.

§Canonical generators

Devela keeps two complementary PRNGs always available:

  • Pcg32 is the canonical general-purpose generator.
  • SplitMix64 is the canonical seed mixer and expander, and also provides a compact 64-bit random stream.

Other generators cover specialized, small-state, legacy, and comparative uses. Their RandQualities indicate known limitations and suitability.

All PRNG types implement Copy. Copying a PRNG duplicates its current state, so both copies produce the same stream until advanced differently.

Some generators are retained for small-device, legacy, educational, or comparative uses despite known generator-side limitations. These are classified with RandQualities::WEAK_PRNG.

Default and ConstInit use fixed seeds and are therefore reproducible. Use explicit or externally sourced seed material when distinct streams are required.

§Seed representation

Byte-backed seeds interpret multi-byte values in little-endian order. This keeps equal seed bytes reproducible across target endianness.

§Features

  • Pcg32 and SplitMix64 are always available.
  • The rand feature enables the extended PRNG palette and generator macros.
  • The std feature enables StdRand.
