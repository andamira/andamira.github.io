Expand description

🎲 Fallible source of raw random data.

📍 num/prob/rand

RandTry is the core trait for random sources that may fail, such as OS calls, hardware RNGs, host APIs, finite buffers, and fallible adapters.

Source behavior and suitability are described by RandQualities.

Infallible sources use Infallible as their error type.

Required Associated Constants§

Source

const RAND_OUTPUT_BITS: u32

Number of bits produced per output step.

Adapters or composite sources may report a conservative value.

Source

const RAND_STATE_BITS: u32

Size of the source state owned by this value, in bits.

External entropy sources that do not store generator state in the value itself should use 0.

Source

const RAND_QUALITIES: RandQualities

Semantic qualities of this random source.

Source

fn rand_try_roll(&mut self, sides: u64) -> Result<u64, Self::Error> ⓘ

Attempts to roll a fair die with a number of sides.

Source

fn rand_try_shuffle<T>(&mut self, slice: &mut [T]) -> Result<(), Self::Error> ⓘ

Attempts to shuffle the elements of the slice in place.

Uses the Fisher–Yates algorithm.

fn rand_try_choose_reservoir<I, F>( &mut self, iter: I, valid: F, ) -> Result<Option<I::Item>, Self::Error> ⓘ
where I: IntoIterator, F: FnMut(&I::Item) -> bool,

Attempts to choose one valid item using reservoir sampling.

This is uniform over all valid items and uses one pass.

Source

fn rand_try_choose_scored<I, F>( &mut self, iter: I, valid: F, ) -> Result<Option<I::Item>, Self::Error> ⓘ
where I: IntoIterator, F: FnMut(&I::Item) -> bool,

Attempts to choose one valid item using random scores.

Each valid item receives a random score. The highest score wins. Ties are resolved uniformly.
