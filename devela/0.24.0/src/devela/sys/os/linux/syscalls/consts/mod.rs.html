35/// File offset type (always 64-bit in modern Linux, even on 32-bit architectures)
36///
37/// Note: While libc's `off_t` may be 32-bit on some 32-bit systems without LFS,
38/// the Linux kernel ABI always uses 64-bit offsets for syscalls. This matches the
39/// behavior of `-D_FILE_OFFSET_BITS=64` in userspace.
40pub(crate) type LinuxOffset = i64;
